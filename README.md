# Python Semantic Cache

A semantic cache solution that stores ChatGPT responses and returns cached results for similar queries, reducing API costs and latency.

## About

This side project aims to create a solution to cache results generated by ChatGPT. When you ask something similar, it returns the response from cache instead of calling the API again.

The project has compatibility with OpenAI, so there's no need to make big changes on your AI agent, AI chatbot, or automation using ChatGPT.

## Benefits

- **Cost Reduction**: No need to send every question to ChatGPT, similar queries return cached responses. PS: save money, because no need to interact everytime with Chatgpt
- **Faster Responses**: Returns cached responses from Redis, which is known for being extremely fast

## Features

- Semantic similarity matching for cache hits
- Easy integration with existing OpenAI code
- Redis-backed storage for persistence
- Configurable similarity threshold

## Installation

Create a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Set your OpenAI API key:

```bash
export OPENAI_API_KEY=your_api_key_here
```

Or create a `.env` file:

```
OPENAI_API_KEY=your_api_key_here
```

## Quick Start

```python
from semantic_cache import SemanticCache

cache = SemanticCache(threshold=0.9)

# First call - hits API
response = cache.get_or_set("What is Python?", lambda: openai.ChatCompletion.create(...))

# Similar question - returns cached response
response = cache.get_or_set("What is Python programming language?", lambda: openai.ChatCompletion.create(...))
```
